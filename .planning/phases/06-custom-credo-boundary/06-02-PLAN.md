# Plan 06-02: Fresh-Implementation Checks (LINT-01, 05, 07, 08, 09, 11)

## Goal

Implement the six remaining LINT checks that require new AST-walking logic. These are more nuanced than Plan 01's checks — some need module-context awareness (LINT-01 allowlisting), AST counting (LINT-05), or pattern matching on nested function call shapes (LINT-11).

## Check Details

### LINT-01: NoRawSwooshSendInLib
- **Pattern**: Any call to `Swoosh.Mailer.deliver/1`, `Swoosh.Mailer.deliver!/1`, or `Swoosh.Mailer.deliver_many/1` in `lib/mailglass/` source files
- **Allowlist**: `Mailglass.Adapters.Swoosh` (the bridge module that legitimately wraps Swoosh)
- **Edge case**: `defmodule MyMailer do; use Swoosh.Mailer; end` pattern in test fixtures should not trigger — check only applies to `lib/` path (configurable via params)

### LINT-05: NoOversizedUseInjection
- **Pattern**: Any `defmacro __using__/1` whose quoted body exceeds a configurable line threshold (default: 20)
- **Counting**: Count AST nodes of type `{:def, _, _}`, `{:defmacro, _, _}`, `{:@, _, [{:behaviour, _, _}]}`, `{:import, _, _}`, `{:alias, _, _}`, `{:require, _, _}`, `{:use, _, _}` in the returned `quote do ... end` block
- **Scope**: Only checks modules under `Mailglass.*` namespace (not adopter code)

### LINT-07: NoDefaultModuleNameSingleton
- **Pattern**: `GenServer.start_link(module, init_arg, name: __MODULE__)` or `GenServer.start_link(module, init_arg, [name: __MODULE__])` — matches the keyword list shape in AST
- **Also matches**: `Agent.start_link/2`, `Registry.start_link/1` with `name: __MODULE__`
- **Allowlist**: None — all library processes must accept a `:name` option

### LINT-08: NoCompileEnvOutsideConfig
- **Pattern**: `Application.compile_env/2`, `Application.compile_env/3`, `Application.compile_env!/2` calls
- **Allowlist**: `Mailglass.Config` module only
- **AST shape**: `{{:., _, [{:__aliases__, _, [:Application]}, :compile_env | :compile_env!]}, _, _args}`

### LINT-09: NoOtherAppEnvReads
- **Pattern**: `Application.get_env/2,3`, `Application.fetch_env/2`, `Application.fetch_env!/2` where the first argument is an atom literal other than `:mailglass`
- **Scope**: Only `lib/mailglass/` files (test code may legitimately read other app env)

### LINT-11: NoFullResponseInLogs
- **Pattern**: `Logger.info/1,2`, `Logger.warning/1,2`, `Logger.error/1,2` calls containing `inspect(response)`, `inspect(resp)`, `inspect(body)`, or string interpolation of `response`/`resp`/`body` variables
- **Heuristic**: Matches variable names containing "response", "resp", "body", "payload" inside `inspect/1` or `#{}` interpolation within Logger calls
- **Level filter**: Only flags `:info`, `:warning`, `:error` — `:debug` level is allowed (off in prod)
