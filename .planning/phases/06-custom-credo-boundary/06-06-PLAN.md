# Plan 06-06: Custom Credo Gate Closure (Gap Plan)

## Goal

Close the Phase 06 blocker by making the custom gate green in real repository state:

`mix credo --strict --only Mailglass.Credo`

`06-VERIFICATION.md` shows the failure is not missing check modules/tests; it is unresolved runtime-repo warnings concentrated in `LINT-03`, `LINT-04`, `LINT-07`, and `LINT-12`. This plan resolves those warnings by combining:

1. **check precision fixes** (eliminate false positives that dilute signal), and
2. **codebase conformance fixes** (remove true positives in `lib/mailglass/**`).

## Gap Inventory (from current custom-Credo run)

- `LINT-03` (`NoUnscopedTenantQueryInLib`) warnings on `Repo.one/get/all/insert` in `Events`, `Outbound`, `SuppressionStore.Ecto`, `Webhook.Ingest`.
- `LINT-04` (`NoBareOptionalDepReference`) warnings for direct `Oban.insert/insert_all` in `Outbound` plus test-only direct `Oban.with_testing_mode/up`.
- `LINT-07` (`NoDefaultModuleNameSingleton`) warnings for `name: __MODULE__` in:
  - `Mailglass.Adapters.Fake.Storage`
  - `Mailglass.RateLimiter.TableOwner`
  - `Mailglass.SuppressionStore.ETS.TableOwner`
- `LINT-12` (`NoDirectDateTimeNow`) warnings in library runtime modules and many test files.

## Execution Plan

### Task 1 — Harden check precision so findings map to real policy risk

1. Update `.credo.exs` parameters and check implementations so:
   - `LINT-04` and `LINT-12` run against `lib/mailglass/**` (policy target) instead of test fixtures.
   - `LINT-03` default `repo_functions` list is restricted to read/query leak vectors (`:all`, `:one`, `:get`, `:get!`, `:get_by`, `:get_by!`) unless explicit unscoped bypass is present.
   - `LINT-03` treats `scope: :unscoped` as valid only when a same-function tenant-bypass audit telemetry emit is present (direct emit or approved helper call).
2. Extend unit tests to lock these boundaries:
   - bad lib file path still flags;
   - test/support path does not flag for LINT-04/LINT-12;
   - LINT-03 flags `scope: :unscoped` without audit emit/helper and allows `scope: :unscoped` with audit emit/helper.
3. Update integration fixture (`test/mailglass/credo/integration_test.exs`) params so synthetic checks execute with the same path/function constraints used in `.credo.exs`.

### Task 2 — Fix true-positive library violations

1. Route direct optional-dep calls through gateway wrappers:
   - add wrapper functions in `Mailglass.OptionalDeps.Oban` for the calls currently used by library runtime paths (`insert`, `insert_all`; include helper for test mode if needed by test helpers);
   - replace direct `Oban.insert/insert_all` in `lib/mailglass/outbound.ex` with gateway calls.
2. Replace direct wall-clock calls in runtime library modules:
   - switch `DateTime.utc_now/0` to `Mailglass.Clock.utc_now/0` in:
     - `lib/mailglass/compliance.ex`
     - `lib/mailglass/events/reconciler.ex`
     - `lib/mailglass/suppression_store/ecto.ex`
3. Remove default singleton registration (`name: __MODULE__`) by plumbing explicit `:name`:
   - `lib/mailglass/adapters/fake/storage.ex`
   - `lib/mailglass/rate_limiter/table_owner.ex`
   - `lib/mailglass/suppression_store/ets/table_owner.ex`
4. Resolve remaining true-positive tenant-scope read paths:
   - update read queries in `lib/mailglass/outbound.ex` and `lib/mailglass/webhook/ingest.ex` to pass through `Mailglass.Tenancy.scope/2` where applicable;
   - where intentionally global/system-level read is required, annotate with explicit `scope: :unscoped` **and** add companion tenant-bypass audit telemetry emit/helper in the same function to satisfy TENANT-03 policy.

### Task 3 — Regression-lock the gate

1. Re-run all custom check unit tests and integration test.
2. Re-run compile lanes and boundary test to ensure no regressions from start_link/clock/tenant changes.
3. Re-run strict custom-Credo command and require zero warnings.

## Trust Boundaries

| Boundary | Why it matters |
|----------|----------------|
| Tenant-scoped read paths (`Repo.*` over tenanted schemas) | Missing scope can leak cross-tenant data (`TENANT-03`). |
| Optional dependency boundary (`Oban` direct calls) | Bypassing gateways breaks degraded-mode guarantees and can cause undefined-module crashes when optional deps are absent. |
| Time source boundary (`DateTime.utc_now/0`) | Non-deterministic time in runtime paths undermines replay and idempotency guarantees under failure handling. |
| Process registration boundary (`name: __MODULE__`) | Hard-coded singleton names in library code can cause collisions in host apps and multi-instance test/runtime scenarios. |
