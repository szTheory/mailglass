# Plan 06-01: Proven-Convention Checks (LINT-02, 04, 06, 10, 12)

## Goal

Implement the five Credo checks for conventions already enforced by code review in Phases 1–4. These checks have zero design risk — the rules are proven and the existing codebase already complies. This plan establishes the `lib/mailglass/credo/` directory, the test harness pattern, and validates the Credo check authoring approach before the more complex checks in Plans 02–03.

## Approach

Each check follows the same skeleton:
1. `use Credo.Check` with `category: :warning` and `explanations: [check: ...]`
2. `def run(%SourceFile{} = source_file, params)` entry point
3. AST walk via `Credo.Code.prewalk/2` or `Macro.postwalk/2` on the source file's AST
4. Pattern-match the specific AST node shape that violates the rule
5. Return `Credo.Issue` structs with `:message`, `:trigger`, `:line_no`, `:column`

Tests use `Credo.SourceFile.parse/2` to create synthetic source from string, then assert on `run/2` output.

## Check Details

### LINT-02: NoPiiInTelemetryMeta
- **Pattern**: `:telemetry.execute/3` or `:telemetry.span/3` where the metadata map argument contains any blocked key atom (`:to`, `:from`, `:body`, `:html_body`, `:subject`, `:headers`, `:recipient`, `:email`, `:cc`, `:bcc`, `:text_body`)
- **AST shape**: `{{:., _, [{:__aliases__, _, [:telemetry]}, :execute]}, _, [_event, _measurements, meta_map]}` where `meta_map` is a map literal containing a blocked key
- **Edge case**: Dynamic map construction (`Map.put(meta, :to, ...)`) is NOT flagged (would require data-flow analysis; out of scope for compile-time lint)

### LINT-04: NoBareOptionalDepReference
- **Pattern**: Any module call to `Oban.*`, `OpenTelemetry.*`, `Mjml.*`, `GenSmtp.*`, `Sigra.*` outside the corresponding `Mailglass.OptionalDeps.*` gateway module
- **AST shape**: `{:., _, [{:__aliases__, _, [:Oban | ...]}, _function]}` where the source file's module is not in the gateway allowlist
- **Allowlist**: `Mailglass.OptionalDeps.Oban`, `Mailglass.OptionalDeps.OpenTelemetry`, `Mailglass.OptionalDeps.MJML`, `Mailglass.OptionalDeps.GenSmtp`, `Mailglass.OptionalDeps.Sigra`

### LINT-06: PrefixedPubSubTopics
- **Pattern**: `Phoenix.PubSub.broadcast/3` or `Phoenix.PubSub.broadcast!/3` where the topic argument is a string literal not starting with `"mailglass:"`
- **Also flags**: `Phoenix.PubSub.subscribe/2` with bare topics (consistency)

### LINT-10: TelemetryEventConvention
- **Pattern**: `:telemetry.execute/3` where the event list doesn't match `[:mailglass, _, _, _ | _]` (must have at least 4 segments, first must be `:mailglass`)
- **Note**: The 5th segment (`:start`/`:stop`/`:exception`) is optional — `Mailglass.Telemetry.span/3` appends it

### LINT-12: NoDirectDateTimeNow
- **Pattern**: `DateTime.utc_now()` or `DateTime.utc_now/0` call outside `Mailglass.Clock` or `Mailglass.Clock.System`
- **Allowlist**: modules under `Mailglass.Clock.*` namespace
