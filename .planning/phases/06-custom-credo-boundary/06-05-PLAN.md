## CI Workflow Update

Add to `.github/workflows/ci.yml`:

```yaml
credo-custom:
  name: "Custom Credo Checks"
  runs-on: ubuntu-latest
  needs: [compile]
  steps:
    - uses: actions/checkout@<sha>
    - uses: erlef/setup-beam@<sha>
      with:
        otp-version: "27.x"
        elixir-version: "1.18.x"
    - run: mix deps.get
    - run: mix credo --strict --only Mailglass.Credo
```

## Integration Test

A single test file with two test groups:
1. **Violation detection**: Synthetic source strings containing one violation each. Assert each check produces exactly one issue.
2. **Clean code**: Synthetic source strings following all conventions. Assert zero issues across all checks.

This provides a regression safety net — if any check's AST matching breaks on a future Elixir version, this test catches it.
