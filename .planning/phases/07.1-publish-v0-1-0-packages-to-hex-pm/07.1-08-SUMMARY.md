# Phase 07.1 Plan 08: Publish Hex Hardening Summary

Hardened the Hex publish workflow so release publishing only runs from the right tag/run, shows package checks before approval, gates on green CI, and serializes core/admin publishing safely.

## What Changed

- Added `workflow_dispatch` with `tag`, `dry_run`, and `package` inputs.
- Replaced the single publish job with four chained jobs: `prepublish-summary`, `gate-ci-green`, `publish-core`, `publish-admin`.
- Added a non-canceling concurrency guard.
- Routed prepublish checks into `$GITHUB_STEP_SUMMARY` before environment approval.
- Added CI status gating on the tagged SHA via `actions/github-script`.
- Added package-aware support to `mix mailglass.publish.check` so the workflow can call it per package.
