  <action>
    Add `Mailglass.Installer.*` modules with concrete signatures:

    - `Mailglass.Installer.Operation`:
      - `@type kind :: :create_file | :ensure_snippet | :ensure_block | :run_task`
      - `defstruct [:kind, :path, :payload, :status, :reason]`
    - `Mailglass.Installer.Plan`:
      - `build(opts, context \\ %{}) :: [Mailglass.Installer.Operation.t()]`
      - include concrete operations for: mail context module, runtime config block, router mounts, default mailable/layout, migration generation task.
      - include conditional Oban worker stub planning using:
        - `oban_available? = Map.get(context, :oban_available?, Mailglass.OptionalDeps.Oban.available?())`
        - when `oban_available?` is `true`, emit `create_file` operation(s) for Oban worker stub template(s)
        - when `oban_available?` is `false`, emit no Oban worker operation and continue install without warnings/errors.
    - `Mailglass.Installer.Apply`:
      - `run(plan, opts) :: {:ok, result_map} | {:error, term}`
      - operation order must be stable (`Enum.sort_by` by `{kind, path}`) before mutation.
    - `Mailglass.Installer.Manifest`:
      - `.mailglass.toml` loader/writer, deterministic hash computation, and lookup by path.
    - `Mailglass.Installer.Conflict`:
      - sidecar writer `write_sidecar(target_path, reason, proposed_contents, opts)`.
      - sidecar naming prefix literal `.mailglass_conflict_`.
    - `Mailglass.Installer.Templates`:
      - templating helpers for owned file output and shared-file managed block bodies.
      - include explicit template helper(s) for generated Oban worker stub content.

    Enforce these concrete behaviors in apply path:
    - installer-owned files: create-or-conflict (unless `--force`)
    - shared files: anchor/block patch only
    - unsafe drift: write sidecar and mark `conflict` status
    - no silent overwrite branches
  </action>
