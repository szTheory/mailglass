## Spacing Scale

Declared values (multiples of 4, mapping to Tailwind v4 utilities):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon inner padding, badge inline gaps |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, tab bar item padding, form label-to-input gap |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card padding, form field stacking gap |
| lg | 24px | `p-6` / `gap-6` | Sidebar section padding, pane header padding |
| xl | 32px | `p-8` / `gap-8` | Layout-level padding (main pane inset) |
| 2xl | 48px | `p-12` / `gap-12` | Major section breaks |
| 3xl | 64px | `p-16` / `gap-16` | Page-level vertical rhythm (reserved; may not appear in v0.1) |

Exceptions:

- **Touch targets**: all interactive sidebar entries, tab buttons, and toggle buttons must
  be a minimum of 44px tall on mobile (WCAG 2.5.5 Level AAA; brand book §12). Use
  `min-h-11` (44px) on tap targets even when visual content is smaller.
- **Device toggle buttons** (375 / 768 / 1024): fixed-width 48px × 32px buttons in a
  segmented control group to keep the three labels scannable side-by-side.
- **Iframe container**: width driven by `@device_width` assign (375 / 768 / 1024px) —
  not a spacing token, a dynamic data-driven CSS inline style.

Source: [DEFAULT 8-point grid; brand book §7.5 for touch target guidance; CONTEXT D-16 for device widths]

## Typography

All sizes in px (rendered via Tailwind v4 `text-*` utilities). Exactly 4 sizes and 2 weights are declared; no other sizes or weights may be used in Phase 5 components.

### Font loading

Load exactly 2 weights per family in `@font-face` declarations:

- **Inter**: 400 (regular), 700 (bold)
- **Inter Tight**: 400 (regular), 700 (bold)
- **IBM Plex Mono**: 400 (regular), 700 (bold)

Do NOT load 500 (medium) or 600 (semibold) for any family. [CONTEXT D-21; BRAND §7.4]

### Typography scale

| Role | Family | Size | Weight | Line Height | Tailwind | Usage |
|------|--------|------|--------|-------------|----------|-------|
| Display | Inter Tight | 20px | 700 | 1.2 | `text-xl font-bold` | Mailable name in main pane header, sidebar "Mailers" heading |
| Heading | Inter Tight | 16px | 700 | 1.3 | `text-base font-bold` | Tab section headers, assigns form group headings, sidebar mailer group headings |
| Body | Inter | 14px | 400 | 1.5 | `text-sm font-normal` | Sidebar scenario names, form labels, metadata, general UI text |
| Label / small / code | Inter (labels) / IBM Plex Mono (code) | 12px | 400 | 1.4–1.6 | `text-xs font-normal` | Badges, status indicators, mailable module namespace labels; code snippets in inline text; `font-mono text-xs` for code contexts |

> **IBM Plex Mono assignment:** IBM Plex Mono renders legibly at 12px in the reference
> browser targets (Chrome/Firefox/Safari on macOS, Chrome/Firefox on Linux). 12px mono
> is used for `<pre>` blocks (Text tab, Raw tab, Headers key/value cells, error stacktraces)
> and inline `<code>` elements. This collapses the previous 13px custom size, dropping
> the non-standard `text-[13px]` utility entirely.

Typography rules:
- **Sentence case** throughout — never ALLCAPS for headings or buttons.
- **Tracking**: no extra letter-spacing on body or headings. Display/Heading use `tracking-tight` (`letter-spacing: -0.02em`) per brand book §7.4 ("slightly condensed in headlines").
- **Glass (#277B96) must never appear as body text on Paper (#F8FBFD)** — contrast ratio 4.8:1 passes AA for large text (18px+ or bold 14px+) but fails for small body text. All body text uses Ink or Slate. [RESEARCH §Accessibility]
- **Max line length** on any prose block (error messages, empty states): `max-w-prose` (≈65ch).
- **Active tab label**: `font-bold` (700) — was `font-medium` (500) in prior draft; weight 500 is not loaded.
- **Bold emphasis within mono blocks**: use `font-bold` (700) — was `font-semibold` (600) in prior draft; weight 600 is not loaded.

Source: [BRAND §7.4 typography hierarchy; RESEARCH §Brand Implementation §Typography]

### Dark theme

The dark chrome toggle (D-16) switches the surrounding admin UI via `data-theme="mailglass-dark"` on the wrapper element. The preview iframe is **not** inside the theme-scoped element — email content renders with its own styles unaffected. Dark theme is admin chrome only at v0.1.

### Visual DON'Ts (enforced in all components)

- No `backdrop-filter: blur()` — no glassmorphism
- No `box-shadow: inset` — no bevels
- No gradients on functional elements (buttons, tab bars, sidebar items)
- No lens flares, literal broken glass visuals, or frosted overlays
- `--depth: 0` and `--noise: 0` on both daisyUI themes
- Never Glass as a large background area

Source: [BRAND §7.4, §11; RESEARCH §Brand Implementation §Visual DON'Ts]

## Interaction Contract

### State machine: no mailable selected → mailable selected → scenario selected

1. **Index (`/dev/mail/`)**: sidebar visible, main pane shows placeholder copy ("Select a scenario from the sidebar to preview it."). No assigns form. No tabs.
2. **Mailable selected with scenarios (`/dev/mail/:mailable`)**: redirect to first scenario automatically.
3. **Scenario selected (`/dev/mail/:mailable/:scenario`)**: assigns form populated with `preview_props/0` defaults; render fires automatically; HTML tab shown by default; tabs active; device + dark toggles active.
4. **Assigns form change**: `phx-change="assigns_changed"` → server re-renders → iframe/pre refreshed. No debounce at v0.1 (typing triggers immediate re-render; LiveView batches naturally).
5. **Device toggle**: `phx-click="set_device"` → `@device_width` updated → iframe `style` width updated via LiveView push → no full re-render of email content.
6. **Dark toggle**: `phx-click="toggle_dark"` → `@dark_chrome` flipped → `data-theme` attribute on wrapper updated → no re-render.
7. **Tab switch**: `phx-click="set_tab"` → `@active_tab` updated → content area re-renders with the correct artifact; no new Renderer call (all four tab contents pre-computed on render and cached in assigns).
8. **LiveReload broadcast**: re-discover all mailables, re-render current scenario if selected, show flash "Reloaded: {file}".
9. **Reset assigns button**: `phx-click="reset_assigns"` → restore `@current_assigns` from `preview_props/0` defaults → re-render.

### Loading state

No explicit loading spinner at v0.1. Renderer call is synchronous in `handle_event`; LiveView's natural socket latency provides implicit feedback. If render takes >200ms (abnormal), the page simply updates slightly late. Document in MAINTAINING.md: if a mailable takes >200ms to render, profile the template — it's the adopter's problem.

### Error recovery

- `preview_props/0` raises on selection: show error card in main pane, sidebar shows `⚠` badge. Rest of dashboard remains fully interactive.
- `Mailglass.Renderer.render/1` raises on assigns form change: catch in `handle_event`, show error card in place of the preview. Assigns form remains editable so the adopter can fix the input.
- Mailable module unloads (hot code reload edge case): next LiveReload broadcast triggers re-discovery; if module disappears, remove from sidebar without crash.

### Accessibility interactions

- All interactive elements reachable via keyboard (`Tab` key, `Enter`/`Space` to activate).
- `<details>/<summary>` for sidebar groups: native keyboard expand/collapse.
- Device toggle uses `role="group"` with `aria-pressed` on each button.
- Tab bar uses `role="tablist"` / `role="tab"` / `aria-selected`.
- Focus ring: `outline: 2px solid var(--color-primary); outline-offset: 2px` on `:focus-visible` for all interactive elements.
- Flash messages: `role="status"` / `aria-live="polite"`.
- Iframe: `title="Email HTML preview"` attribute.

## Asset Pipeline

Not a design contract per se, but recorded here so the checker can confirm no Node toolchain is implied by design decisions:

- **Build tool**: `{:tailwind, "~> 0.4"}` Hex package (Tailwind v4.1.12 bundled binary)
- **No esbuild**: zero custom JS at v0.1
- **CSS input**: `mailglass_admin/assets/css/app.css`
- **CSS output**: `mailglass_admin/priv/static/app.css` (minified, committed to git)
- **Font files**: `priv/static/fonts/*.woff2` (committed, pre-subsetted; Inter 400+700, Inter Tight 400+700, IBM Plex Mono 400+700)
- **Serving**: `MailglassAdmin.Controllers.Assets` compile-time `File.read!` + MD5 hash (LiveDashboard pattern)
- **Adopter install**: zero endpoint.ex edits, zero Node toolchain
