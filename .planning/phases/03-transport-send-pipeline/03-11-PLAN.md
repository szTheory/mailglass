<objective>
Fix the citext OID cache flake surfaced by Phase 3's full-suite runs. 30 test
failures trace through `suppression_store/ecto.ex:55` with
`Postgrex.Error — cache lookup failed for type X`. Root cause: Postgrex caches
citext OIDs at connection open time; when `mix ecto.drop && mix ecto.create`
recreates the DB, citext gets a new OID, but existing pooled connections still
hold the old OID in cache.

The Plan 02-06 state recorded a partial mitigation:
`config/test.exs: disconnect_on_error_codes: [:internal_error]`
plus a `probe_until_clean/5` helper in `persistence_integration_test.exs`.

This plan confirms the mitigation is in place and consistent, adds any
missing config, and ensures the TestRepo startup sequence in test_helper.exs
flushes stale connections before the suite begins.

Purpose: 30 intermittent failures drown signal in CI and make the Phase 3
UAT gate unreliable. A one-time config fix eliminates the entire class.

Output:
- config/test.exs — verified/updated disconnect_on_error_codes
- test/test_helper.exs — verified/updated TestRepo probe sequence
</objective>

<!-- Postgrex config pattern: -->
In config/test.exs for the TestRepo:
```elixir
config :mailglass, Mailglass.TestRepo,
  # ... other opts ...
  disconnect_on_error_codes: [:internal_error]
```

`disconnect_on_error_codes: [:internal_error]` instructs Postgrex to drop and
reconnect the pool connection whenever a Postgres `internal_error` (SQLSTATE XX000)
is returned. Type OID cache misses return XX000. On reconnect the connection
re-registers all types, picking up the new citext OID.

<!-- test_helper.exs probe pattern from Plan 02-06: -->
After starting the TestRepo and running migrations, execute one probe query
that forces a citext comparison. If it raises Postgrex.Error (XX000), the
disconnect_on_error_codes config fires and the next call succeeds. A simple
`TestRepo.query!("SELECT 1::citext")` or similar does the job.
