       ```elixir
       defmodule Mailglass.Webhook.Router do
         @moduledoc """
         Router macro for mounting Mailglass webhook endpoints in your
         adopter Phoenix router. Mirrors the `Phoenix.LiveDashboard.Router`
         + `Oban.Web.Router` idiom — used inside an adopter-owned `scope`
         block with adopter-owned `pipe_through` (CONTEXT D-06).

         ## Usage

             defmodule MyAppWeb.Router do
               use Phoenix.Router
               import Mailglass.Webhook.Router

               pipeline :mailglass_webhooks do
                 plug :accepts, ["json"]
                 # NO :browser, :fetch_session, :protect_from_forgery — webhooks have none
               end

               scope "/", MyAppWeb do
                 pipe_through :mailglass_webhooks
                 mailglass_webhook_routes "/webhooks"
               end
             end

         Generates two `post` routes by default:

           * `POST /webhooks/postmark` → `Mailglass.Webhook.Plug` `[provider: :postmark]`
           * `POST /webhooks/sendgrid` → `Mailglass.Webhook.Plug` `[provider: :sendgrid]`

         ## Options

           * `:providers` — list of `:postmark | :sendgrid` (default both)
           * `:as` — route helper prefix (default `:mailglass_webhook`)

         The provider list is locked at v0.1 (PROJECT D-10 defers Mailgun/SES/Resend
         to v0.5). v0.5 will extend the validated set without breaking v0.1
         adopters who pass the default.

         ## Endpoint wiring (separate from this macro)

         Configure `Plug.Parsers` in your endpoint with mailglass's
         `CachingBodyReader`:

             plug Plug.Parsers,
               parsers: [:json],
               pass: ["*/*"],
               json_decoder: Jason,
               body_reader: {Mailglass.Webhook.CachingBodyReader, :read_body, []},
               length: 10_000_000   # 10 MB cap with 2 MB headroom

         The 10 MB cap accommodates SendGrid batches up to 128 events.
         """

         @valid_providers [:postmark, :sendgrid]
         @default_providers @valid_providers
         @default_as :mailglass_webhook

         @doc """
         Generate provider-per-path POST routes for `Mailglass.Webhook.Plug`.

         Must be invoked inside a `Phoenix.Router` `scope` block. See
         the module doc for full usage.

         ## Options

           * `:providers` — defaults to `#{inspect(@default_providers)}`
           * `:as` — defaults to `#{inspect(@default_as)}`
         """
         defmacro mailglass_webhook_routes(path, opts \\ []) do
           providers = Keyword.get(opts, :providers, @default_providers)
           as = Keyword.get(opts, :as, @default_as)

           # Validate at compile time — fails at router-mount time, not request time
           Enum.each(providers, fn p ->
             unless p in @valid_providers do
               raise ArgumentError,
                     "Mailglass.Webhook.Router: unknown provider #{inspect(p)} " <>
                       "(valid at v0.1: #{inspect(@valid_providers)}; " <>
                       "Mailgun/SES/Resend land in v0.5)"
             end
           end)

           quote bind_quoted: [path: path, providers: providers, as: as] do
             for provider <- providers do
               post "#{path}/#{provider}",
                 Mailglass.Webhook.Plug,
                 [provider: provider],
                 as: :"#{as}_#{provider}"
             end
           end
         end
       end
       ```

       ```elixir
         like `mailglass_webhook_routes "/tenants/:tenant_id/webhooks"`
         (custom path with embedded tenant) compose this resolver alongside
         their own `scope/2` impl.

         ## Why this is a separate module

         `Mailglass.Tenancy.SingleTenant` is the zero-config default
         (always `{:ok, "default"}`). `ResolveFromPath` is the minimum
         viable multi-tenant resolver that adopters can wire by setting:

             config :mailglass, tenancy: Mailglass.Tenancy.ResolveFromPath

         ## Important: this module does NOT implement `scope/2`

         It is a SUGAR resolver that handles ONLY `resolve_webhook_tenant/1`.
         Adopters using it must pair with a Tenancy impl that handles
         `scope/2` (typically their own per-tenant Repo prefix / WHERE
         clause logic). The Webhook plug only needs `resolve_webhook_tenant/1`;
         the rest of the persistence pipeline calls `scope/2`.
         """

         @behaviour Mailglass.Tenancy

         @impl Mailglass.Tenancy
         def scope(_query, _context) do
           raise """
           Mailglass.Tenancy.ResolveFromPath does not implement scope/2.

           This module is a sugar resolver for webhook tenant extraction
           only. To use it for the full Tenancy contract, configure your
           own module that delegates `resolve_webhook_tenant/1` to this
           module while implementing `scope/2` for your data layer.
           """
         end

         @impl Mailglass.Tenancy
         def resolve_webhook_tenant(%{path_params: %{"tenant_id" => tid}}) when is_binary(tid) and tid != "" do
           {:ok, tid}
         end

         def resolve_webhook_tenant(_context), do: {:error, :missing_path_param}
       end
       ```
