# v0.1 Milestone Audit

**Audited:** 2026-04-25
**Status:** `gaps_found`
**Supersedes:** 2026-04-22 audit (stale — phases 3-7 had not executed)

This audit is a fresh end-to-end review against the live codebase. Phases 1, 2, 3, 4, 6 each have a `passed` VERIFICATION.md. Phases 5 and 7 are missing VERIFICATION.md altogether — retroactively verified here against ROADMAP.md success criteria.

## Verdict

**Do not close v0.1 yet.** Three installer blockers (G-1, G-2, G-3) make the headline marketing claim — "zero to first preview-styled HEEx email in under 5 minutes via `mix mailglass.install` + 5 lines of mailable code" — false in practice. An adopter following the documented success criterion would hit `UndefinedFunctionError` at router compile time. The defect ships behind a green CI gate because the golden-diff fixture (G-2/G-5) drives a simulated install path rather than the real `Plan.build` + `Apply.run` output.

A paired Phase 7.1 (insertion) is the cleanest path:
1. Rewrite `Mailglass.Installer.Templates.router_mount_snippet/1` to emit `import MailglassAdmin.Router` + `mailglass_admin_routes "/dev/mail"`.
2. Add a webhook-mount snippet (`import Mailglass.Webhook.Router; mailglass_webhook_routes [:postmark, :sendgrid]`) and the matching `Plug.Parsers body_reader: {Mailglass.Webhook.CachingBodyReader, …}` endpoint config block.
3. Drop `run_simulated_install!/2` from the test fixtures; drive `Mailglass.Installer.Apply.run/2` directly so the golden snapshot reflects what adopters actually receive.
4. Regenerate `test/example/README.md` against the real installer output.
5. Tag at least one mailglass_admin smoke test with `@tag :admin_smoke` so the existing `admin_smoke_gate` CI job matches a real test rather than passing vacuously (G-4).

Once those land, write VERIFICATION.md for Phases 5 and 7, flip the ~25 stale traceability rows in REQUIREMENTS.md, and `/gsd-complete-milestone v0.1` is unblocked.

## Cross-Phase Wiring Gaps

### Blockers

**G-1** — `lib/mailglass/installer/templates.ex:33` emits `{App}.MailglassAdmin.Router.mount()`, a non-existent API. Real Phase 5 macro is `mailglass_admin_routes/2`.

**G-2** — `test/support/installer_fixture_helpers.ex:11` hardcodes `forward "/dev/mailglass", MailglassAdmin.Router` (wrong shape — Router is a macro module). Golden snapshot captured against this fixture rather than real `Apply.run` output.

**G-3** — `lib/mailglass/installer/plan.ex` generates preview-router snippet but never emits webhook-router mount or `Plug.Parsers body_reader: {CachingBodyReader, …}` endpoint wiring. First adopter webhook POST raises `%ConfigError{type: :webhook_caching_body_reader_missing}`.

**G-5** — `installer_fixture_helpers.ex:34-39` falls back to `run_simulated_install!/2` in test contexts; real installer is never exercised by golden test, so G-1/G-2/G-3 ship unblocked by CI.

### Tech debt

**G-4** — `.github/workflows/ci.yml:360` runs `mix test --only admin_smoke` but no test in `mailglass_admin/test/` carries `@tag :admin_smoke`. Job passes vacuously.

**G-6** — `MailglassAdmin.OptionalDeps.PhoenixLiveReload` gateway exists but its `available?/0` is unused — only consumer is a `Code.ensure_loaded?` short-circuit. Cosmetic divergence from `Mailglass.OptionalDeps.{Oban, …}` pattern.

**Phase 4 standard-depth review WR-01..WR-06** — see `.planning/phases/04-webhook-ingest/04-REVIEW.md`. None block the goal.
